# TEYCA — Управление клиентами и PUSH-рассылками

Веб-приложение для работы с клиентской базой электронных карт лояльности TEYCA. Позволяет авторизоваться, просматривать список клиентов с поиском и пагинацией, а также отправлять PUSH-уведомления выбранным клиентам.

---

| RxJS | 7.8 | Асинхронные потоки (HTTP, debounce) |

## Описание каждого файла

### 1. Точка входа

#### `src/main.ts`
Запускает Angular-приложение. Вызывает `bootstrapApplication` со standalone-компонентом `AppComponent` и конфигурацией `appConfig`. Никаких NgModule — используется современный подход Angular 19.

#### `src/index.html`
HTML-страница с подключением шрифтов Roboto и Material Icons через Google Fonts. Содержит `<app-root>` — корневой элемент, в который Angular рендерит приложение.

#### `src/styles.scss`
Глобальные стили: сброс margin у body, подключение шрифта, кастомный скроллбар. Тема Angular Material подключена через `angular.json` (prebuilt-themes/azure-blue.css).

---

### 2. Конфигурация приложения

#### `src/app/app.config.ts`
Центральная конфигурация Angular. Здесь регистрируются все провайдеры:
- **`provideRouter`** — маршрутизация с `withComponentInputBinding()` (можно привязывать route params к @Input)
- **`provideHttpClient`** — HTTP-клиент с перехватчиком `authInterceptor`
- **`provideAnimationsAsync`** — анимации Angular Material (ленивая загрузка)
- **`provideNativeDateAdapter`** — адаптер дат для MatDatepicker

#### `src/app/app.routes.ts`
Маршруты приложения:

| Путь | Компонент | Защита |
|---|---|---|
| `/login` | LoginComponent | Нет |
| `/clients` | ClientsComponent | `authGuard` — только для авторизованных |
| `/` и `**` | Редирект на `/clients` | — |

Компоненты загружаются лениво через `loadComponent` — код страницы подгружается только при переходе на неё.

#### `src/app/app.component.ts`
Корневой компонент — просто `<router-outlet />`. Вся навигация происходит внутри него.

---

### 3. Модели данных (`core/models/`)

#### `pass.model.ts`
Интерфейс `Pass` — полная модель электронной карты клиента. Содержит все поля, которые возвращает TEYCA API: `user_id`, `fio`, `phone`, `email`, `bonus`, `discount`, `barcode` и десятки дополнительных полей (ключи, триггеры, ссылки и пр.).

Также экспортирует `PassTableColumn` — тип для колонок, отображаемых в таблице.

#### `api-response.model.ts`
Интерфейсы для типизации ответов API:
- **`PassesResponse`** — `{ data: Pass[], meta: ApiMeta }` — список карт с метаинформацией о пагинации
- **`ApiMeta`** — `{ size, limit, offset }` — общее кол-во записей, лимит и смещение
- **`AuthResponse`** — `{ token }` — ответ авторизации
- **`PushRequest`** — `{ message, user_ids, date_start? }` — тело запроса на отправку PUSH
- **`ApiErrorBody`** — `{ error?, message? }` — структура ошибки API

#### `index.ts`
Barrel-файл. Реэкспортирует все модели из одной точки, чтобы в импортах писать `from '../models'` вместо отдельных путей.

---

### 4. Сервисы (`core/services/`)

#### `auth.service.ts` — Сервис авторизации
Отвечает за вход/выход и хранение токена.

**Ключевые моменты:**
- **`token`** — Angular Signal. Инициализируется из `localStorage` при старте. Все компоненты, которые его читают, автоматически реагируют на изменения.
- **`isAuthenticated`** — computed-сигнал, `true` если токен есть. Используется в guard и UI.
- **`login()`** — POST на `https://api.teyca.ru/test-auth-only`. Сервер принимает любые данные и возвращает тестовый токен. Токен сохраняется в `localStorage` и в signal.
- **`logout()`** — очищает токен и перенаправляет на `/login`.
- **`getApiUrl()`** — строит URL для API-запросов, вставляя токен в path: `/v1/{token}/passes`.

#### `passes.service.ts` — Сервис электронных карт
Один метод `getPasses()` — загружает список клиентов с сервера.

**Параметры запроса:**
- `limit` и `offset` — пагинация (API возвращает макс. 1000 записей)
- `search` — серверный поиск по телефону. Формат: `search=phone=79876543211`

Использует `HttpParams` для безопасной сборки query string.

#### `push.service.ts` — Сервис PUSH-рассылок
Метод `sendPush()` — POST на `/v1/{token}/message/push`.

**Тело запроса:**
```json
{
  "message": "Текст уведомления",
  "user_ids": [123, 456],
  "date_start": "2025-01-15T10:00:00.000Z"  // опционально
}
```
Если `date_start` не указан — рассылка мгновенная.

---

### 5. Interceptor (`core/interceptors/auth.interceptor.ts`)

Функциональный HTTP-перехватчик. Встраивается в цепочку HTTP-запросов и делает две вещи:

1. **Добавляет заголовок авторизации** — ко всем запросам на `api.teyca.ru` добавляет `Authorization: {token}`.
2. **Обрабатывает 401** — если API вернул "не авторизован", вызывает `logout()` и перенаправляет на логин.

Запросы на другие домены (CDN, шрифты и пр.) не трогает.

---

### 6. Guard (`core/guards/auth.guard.ts`)

Функциональный route guard. Вызывается Angular Router перед переходом на защищённый маршрут (`/clients`).

- Если `isAuthenticated() === true` → пропускает.
- Если нет → возвращает `UrlTree` на `/login` (редирект).

---

### 7. Страница входа (`features/auth/login.component.ts`)

Standalone-компонент с формой авторизации.

**Форма (Reactive Forms):**
- `login` — обязательное поле
- `password` — обязательное поле, с кнопкой показа/скрытия пароля

**Логика:**
1. При submit валидирует форму. Если пустые поля — показывает mat-error.
2. Отправляет POST через `AuthService.login()`.
3. При успехе — навигация на `/clients`.
4. При ошибке — показывает `MatSnackBar` с текстом ошибки.

**UI:**
- Карточка по центру экрана на градиентном фоне
- Иконки в полях ввода
- Spinner в кнопке во время загрузки
- Signal `loading` блокирует кнопку во время запроса

---

### 8. Страница клиентов (`features/clients/clients.component.ts`)

Основная рабочая страница. Таблица клиентов с поиском, пагинацией и выбором для PUSH.

**Toolbar:**
- Название приложения
- Кнопка "Выйти" (вызывает `logout()`)

**Поиск:**
- Текстовое поле с debounce 400мс — не отправляет запрос на каждый символ, а ждёт 400мс после последнего ввода
- `distinctUntilChanged` — не запрашивает повторно если значение не изменилось
- Поиск серверный: отправляет `search=phone=...` на API

**Таблица (mat-table):**

| Колонка | Описание |
|---|---|
| Checkbox | Выбор клиентов для PUSH |
| ID | `user_id` |
| ФИО | `fio` |
| Телефон | `phone` |
| E-mail | `email` |
| Бонусы | `bonus` |
| Скидка | `discount` |
| Штрих-код | `barcode` |

**Пагинация (mat-paginator):**
- Серверная — запрашивает только текущую страницу
- `pageSize` = 10/20/50
- При переключении страницы: `offset = pageIndex * pageSize`
- Общее кол-во записей берётся из `meta.size`

**Выбор клиентов:**
- `SelectionModel` из `@angular/cdk` — управляет множественным выбором
- Checkbox "выбрать все" — выделяет всю текущую страницу
- Кнопка "Отправить PUSH" активна только когда выбран хотя бы 1 клиент

**Обработка ошибок:**
- `MatSnackBar` с текстом ошибки от API
- `finalize()` — снимает loading в любом случае (успех или ошибка)

**Angular-фишки:**
- `takeUntilDestroyed(this.destroyRef)` — автоотписка при уничтожении компонента (замена `OnDestroy + Subject`)
- Signals для состояния: `clients`, `totalItems`, `loading`, `pageSize`
- `@if` / `@for` — новый синтаксис шаблонов Angular 19

### 9. Диалог PUSH-рассылки (`features/clients/push-dialog.component.ts`)

Модальное окно (MatDialog) для отправки PUSH-уведомлений.

**Поля формы:**
- **Текст сообщения** — textarea, обязательное. Показывает счётчик символов.
- **Отложенная отправка** — checkbox. При включении появляются:
  - **Дата** — MatDatepicker (календарь)
  - **Время** — input type="time" (ЧЧ:ММ)

**Логика отправки:**
1. Валидация формы (текст обязателен).
2. Формирует `PushRequest`:
   - `message` — текст
   - `user_ids` — массив ID выбранных клиентов (передаётся через `MAT_DIALOG_DATA`)
   - `date_start` — ISO-строка даты+времени (если отложенная отправка)
3. Если дата не указана — PUSH отправляется мгновенно.
4. При успехе — закрывает диалог с результатом `'sent'`.
5. При ошибке — показывает snackbar.

**Связь с ClientsComponent:**
- Диалог открывается из `ClientsComponent.openPushDialog()`
- Получает `userIds` через `MAT_DIALOG_DATA`
- После закрытия с результатом `'sent'` — родитель показывает уведомление и сбрасывает выбор

## Как это работает (сценарий пользователя)

1. Пользователь открывает приложение → редирект на `/login`
2. Вводит любые логин/пароль → получает токен → редирект на `/clients`
3. Загружается таблица клиентов (первые 20 записей)
4. Можно искать по телефону — поиск серверный с debounce
5. Переключение страниц — запрос на API с новым offset
6. Выбирает клиентов чекбоксами → нажимает "Отправить PUSH"
7. В модальном окне вводит текст, опционально выбирает дату
8. Нажимает "Отправить" → запрос на API → уведомление об успехе
